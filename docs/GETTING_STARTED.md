# プロジェクト作業開始ガイドライン

このドキュメントは、新しい放置ゲームプロジェクトで作業を開始する際の手順をまとめたものです。

## 作業開始時の手順

### 1. ドキュメントの確認

作業開始前に、以下のドキュメントを確認してください:

| ドキュメント         | 内容                                     | 参照するタイミング |
| -------------------- | ---------------------------------------- | ------------------ |
| `DEVELOPMENT_DOC.md` | プロジェクト全体の構造、各ファイルの役割 | 必ず最初に確認     |
| `UI_GUIDELINES.md`   | UI設計方針、レイアウトルール             | UI関連の作業時     |
| `AUTOMATION_DOC.md`  | 自動化機能の仕様                         | 自動化関連の作業時 |

### 2. 作業タイプ別の確認事項

#### 新機能追加

1. `DEVELOPMENT_DOC.md` で関連ファイルを特定
2. `constants.js` のデータ構造を確認
3. 類似機能の実装パターンを参照

#### UI変更

1. `UI_GUIDELINES.md` を必ず確認
2. 既存UIの実装パターンを `index.html` で確認
3. CSSクラスの使用方法を確認

#### バグ修正

1. 関連する `.js` ファイルを特定
2. `game.js` のメインループとの関係を確認

### 3. 主要ファイル

| ファイル       | 役割                         |
| -------------- | ---------------------------- |
| `constants.js` | プレイヤーデータの初期値定義 |
| `game.js`      | メインループ、Vue.jsアプリ   |
| `index.html`   | UI定義                       |
| 各機能の `.js` | 機能ごとのロジック           |

### 4. コーディング規則

- 新しいプレイヤーデータは `constants.js` の `initialData()` に追加
- UI表示は `index.html` に追加
- ロジックは専用の `.js` ファイルに実装
- 既存の命名規則・スタイルに従う

- [ ] 機能が正しく動作する
- [ ] UIガイドラインに従っている
- [ ] 必要に応じてドキュメントを更新した

## AIエージェントへの指示

このプロジェクトで作業する全てのAIエージェントは、以下のルールを遵守してください：

1. **コンテキストの確立**: セッション開始時に必ずこの `GETTING_STARTED.md` を読み、プロジェクトの構造とルールを把握すること。
2. **UIの一貫性**: UIを変更・追加する場合は、必ず `UI_GUIDELINES.md` を参照し、既存のスタイル（特に1行レイアウト）を厳守すること。
3. **データ構造**: チートやバグを避けるため、`constants.js` のデータ構造を勝手に変更せず、既存のパターンに従うこと。
4. **自動化**: 自動化機能に触れる場合は `AUTOMATION_DOC.md` を確認すること。
